# PHASE 3 — OBSERVABILITY

Implement production observability.

Do NOT implement new notification types.

Implement only infrastructure.

---

## 1. Delivery Status Tracking

Every notification must expose lifecycle states.

Example:

PENDING

QUEUED

PROCESSING

SENT

DELIVERED

FAILED

DEAD_LETTERED

BOUNCED (if webhook later exists)

Do not invent unsupported transitions.

---

## 2. Retry History

Persist:

* retry count
* retry timestamps
* retry reason
* retry backoff delay
* final retry outcome

---

## 3. Provider Response Tracking

Persist:

* provider message ID
* provider name
* request timestamp
* response timestamp
* provider status
* provider error code
* provider error message

Never store API keys.

---

## 4. Failure Dashboard

Implement backend support for:

Failed notifications

Grouped by:

* event type
* provider
* error type
* date
* retry status

No frontend required.

Expose reusable service methods.

---

## 5. Manual Resend

Implement backend capability for administrators to resend a failed notification.

Requirements:

* reuse original payload
* preserve audit trail
* create new delivery attempt
* never mutate original delivery record

---

## 6. Search & Filtering

Support filtering notifications by:

* recipient
* event type
* status
* provider
* created date
* delivered date
* failed date
* correlation ID
* request ID

---

## 7. Metrics

Implement service methods producing metrics.

Examples:

* total sent
* total failed
* delivery success rate
* average provider latency
* average retry count
* dead-letter count
* queue latency
* processing latency

Do not implement Grafana.

Do not implement Prometheus exporters.

Only backend metrics.

---

## 8. Audit

Every resend

Every retry

Every provider failure

Every provider success

Every dead-letter

must generate immutable audit records.
