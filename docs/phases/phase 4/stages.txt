One thing I would strongly recommend before implementation starts:

Create:

docs/implementation/booking/implementation-order.md

This prevents the same problem that often happens in large Django projects where APIs, events, and websockets get built before the domain layer is stable.

For scalability and maintainability, the Booking domain should remain:

Models
 ↓
Services
 ↓
Events
 ↓
API / WebSocket / Tasks

and never the reverse.
