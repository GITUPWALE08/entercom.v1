# Quote Flow Workflow

## 1. Purpose
Defines the rules, versioning, and transition logic for generating and approving financial quotes tied to requests. It ensures transparent financial agreements between Entercom and the customer before work commences.

## 2. Scope
- Quote generation by staff and technicians.
- Quote revision limits and customer interactions (approve/reject).
- Expiration tracking and its impact on the request lifecycle.
- Unique inspection workflow sequencing.

## 3. Out of Scope
- Payment gateway integration details and PCI compliance.
- Line-item inventory mapping.
- Quote PDF generation formatting.

## 4. Definitions
*   Refer to `docs/architecture/request/domain-glossary.md` for base definitions.

## 5. Rules
*   **Generation Authority**: Quotes can be generated by technicians or staff.
*   **Inspection Exception**: For the `inspection` category, booking and scheduling MUST occur *before* quote generation (quote is generated post-inspection).
*   **Revision Limit**: A maximum of 3 revisions is allowed per quote. Once exceeded quote expires
*   **Expiry**: Quotes expire 30 days after generation.
*   **Expiry Action**: If a quote expires, the associated request is automatically transitioned to `cancelled`.
*   **Immutability**: Rejected quotes are immutable. A new version is required; old quotes are never overwritten.
*   **Reassignment Lock**: If the technician who created an active quote is reassigned, ONLY staff/managers may revise the existing quote.
*   **Fail-Closed**: If quote approval fails (e.g., signature mismatch), the quote remains unapproved, and the request stays in `awaiting_customer_approval`.

## 6. Required Matrices/Tables

### Quote Transition Matrix
| Customer Action | Allowed Result | Audit Action |
| :--- | :--- | :--- |
| **Approve** | Transition to `awaiting_payment` or `awaiting_assignment` | `quote.approved` |
| **Reject** | Quote locked. Transition to `awaiting_quote` for new version | `quote.rejected` |
| **Request Revision** | Triggers new version generation (counts toward limit) | `quote.revision_requested` |

## 7. Edge Cases
*   **Revision Limit Exceeded**: If a customer requests a 4th revision, the flow is blocked. The quote expires

## 8. Audit Expectations
Every quote interaction must be logged with:
*   `actor`, `action`, `timestamp`, `correlation_id`, `request_id`, `previous_state`, `new_state`, `reason`
*   Financial revisions must log the version number delta.
